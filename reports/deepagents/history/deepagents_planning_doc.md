# Deep Agents 범용 에이전트 하네스 기획서 초안

## 1. 개요 (Overview)

| 항목 | 내용 |
|---|---|
| 프로젝트명 | Deep Agents |
| 비즈니스 목적 | 계획 수립, 파일 시스템 조작, 쉘 실행, 서브 에이전트 위임 등을 포함한 범용 에이전트 하네스 제공 |
| 주요 기능 | 계획(Planning), 파일 읽기/쓰기, 쉘 명령 실행, 서브 에이전트 관리, 자동 컨텍스트 요약 |
| 실행 방식 | Python SDK와 CLI 형태로 사용 가능하며 모델 독립적, 즉시 실행 가능한 에이전트 템플릿 제공 |
| 기대효과 | 빠른 에이전트 개발 및 확장성 확보, 다양한 모델 및 샌드박스 연동 지원 |

## 2. Agent 유형 분류 (Classification)

| 항목 | 내용 |
|---|---|
| Agent 유형 | 범용 AI 에이전트 하네스 (Planning, Tool 사용, Context 관리 포함) |
| 실행 주기 | 사용자 또는 호출 시점에 동작, CLI/SDK 기반 직접 호출 방식 |
| 입력 성격 | 사용자 명령, 메시지 형태의 대화 및 작업 요청 |
| 출력 성격 | 작업 결과 텍스트, 파일 시스템 상태 변경, 쉘 실행 결과 등 |
| 운영 형태 | 로컬 개발환경 및 배포형 서비스(예: LangSmith Deploy)를 통한 자가 호스팅 가능 |

## 3. 업무 Flow 및 구현 방식 (Work Process)

### 에이전트 초기화 및 호출
- SDK 또는 CLI를 통해 에이전트 객체 생성
- 명령 또는 메시지 기반 작업 요청 전달

### 계획 수립 및 작업 분해
- 사용자 요청에 대한 할 일(TODO) 목록 작성
- 진행 상황 및 다음 작업 산출

### 파일 시스템 조작
- 파일 읽기, 쓰기, 편집 함수 실행
- 디렉터리 목록 조회, 파일 검색 등 작업 수행

### 쉘 명령 실행
- 안전하게 제한된 쉘 명령 실행
- 외부 프로세스 호출 및 결과 처리

### 서브 에이전트 위임
- 복잡 작업 분할 시 서브 에이전트 생성
- 서브 에이전트와 독립적 컨텍스트 관리 및 결과 통합

### 컨텍스트 자동 관리
- 대화 및 작업 기록 자동 요약 저장
- 긴 대화 시 컨텍스트 크기 관리

## 4. 운영 Tool 리스트 (Tool List)

| 항목 | 내용 |
|---|---|
| Python SDK (deepagents) | 에이전트 기능 및 내부 로직 구현 |
| CLI 도구 (deepagents-cli) | 터미널 기반 에이전트 실행 및 인터페이스 제공 |
| MCP 서버 (docs-langchain, reference-langchain) | 모델 컨텍스트 프로토콜 활용 문서 및 참조 |
| Sandbox Providers (Daytona, Modal, QuickJS, Runloop 등) | 코드 실행, 격리 환경 제공을 위한 샌드박스 인프라 |
| Make / uv / ruff / ty | 개발 환경에서 의존성 관리, 빌드 자동화, 코드 스타일/정적 분석 |

## 5. Agent 동작 필수 파라미터 (Parameters)

### 환경 및 실행 설정 파라미터
- model: 사용할 LLM 모델 지정 (예: claude-*, gpt-*, gemini-*)
- working_directory: 에이전트 실행/작업 디렉터리 설정 (기본 '.')
- timeout: 최대 실행 시간 제한 (기본 30분)
- shell_allow_list: 허용된 쉘 명령 목록 (예: recommended,git,gh)
- enable_memory: 작업 간 메모리 지속 여부 (기본 true)
- memory_scope: 메모리 캐시 범위 (pr, branch, repo)
- agent_name: 에이전트 고유 이름으로 메모리 네임스페이스 결정

### 확인 필요 파라미터
- skills_repo: 사용할 GitHub Skills 저장소 경로, 기본값 없음 (기능 연동 여부 추정 필요)
- cli_version: deepagents-cli 버전 지정, 생략 시 최신 사용

## 6. 기타 참고사항 및 제약조건

- 에이전트 실행 시 쉘 명령 제한 설정을 통해 보안 확보 필요
- 메모리 지속 기능 활성화 시 개인정보 및 민감데이터 저장 리스크 검토 필요
- 서브 에이전트간 컨텍스트 분리 및 동기화 과정 안정성 검증 필요
- 베타 상태인 'deepagents deploy' 기능 사용 시 API 및 설정 변경 가능성 주의
- 프로젝트 내 다수 패키지로 모노레포 구성, 버전 관리 및 배포 자동화 프로세스와의 연계 필수
- 외부 서비스(API 키, GitHub 토큰 등) 사용 시 권한 관리 및 비밀정보 노출 위험 주의
- Linting과 테스트 자동화 구축되어 있으나, 신규 개발 시 해당 규칙 엄수 필요
- 다양한 모델과 샌드박스 지원으로 확장성 우수하나, 통합 테스트 및 호환성 지속 검토 필요

## Assumptions

- agent 기능은 기본적으로 오픈소스 SDK와 CLI 중심으로 제공됨
- 여러 샌드박스 파트너 연동은 선택적이며, 프로젝트 상태에 따라 미지원 환경 존재 가능
- API 키 및 토큰 입력값은 별도 보안 저장소로 관리해야 하며, 저장소 내 직접 노출 금지
- 실제 배포 시 'deepagents deploy'를 통한 LangSmith Deploy 연동이 권장 방식으로 추정됨
- 다양한 LLM 모델 지원은 MCP 프로토콜 및 Agent Skills 활용으로 대응 중임
- 사용자 커스텀 프롬프트 및 툴 추가는 SDK 확장점으로 제공되며, 기본 하드코딩 X
