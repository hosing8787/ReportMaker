# Deep Agents 에이전트 기획서

- 작성 배경
  - 본 프로젝트는 오픈소스 Python 기반 Deep Agents 저장소를 기반으로 범용 에이전트 하네스를 제공하고자 함.
  - Deep Agents는 계획 수립, 파일시스템 조작, 쉘 명령 실행, 서브에이전트 위임 등 다양한 작업을 수행 가능하도록 설계됨.
  - CLI 및 SDK 형태로 제공되어 사용자가 즉시 활용 가능하도록 문서와 툴링을 포함함.
- 목적
  - 복잡한 작업을 분해하고 추적할 수 있는 계획 기능 제공.
  - 파일 읽기, 쓰기, 편집, 검색 등 파일시스템과의 원활한 인터페이스 제공.
  - 쉘 명령 실행을 통한 외부 명령어 호출 및 작업 환경 조성.
  - 서브에이전트를 통한 작업 위임 및 병렬 처리 가능.
  - 자동 요약 및 큰 출력 처리 등 대화 컨텍스트 관리 기능 내장.
  - 모델 및 툴 교체, 확장이 용이한 모듈화 구조 제공.
- 비고
  - 확인된 정보 기준으로 작성, 미확정 사항은 '확인 필요'로 명시함.
  - 스크린샷 양식에 맞춰 AS-IS/TO-BE 및 구현방안 구조 유지.
  - 기능 설명은 간결명료하게 요약, 내부 세부 구현은 필요시 '확인 필요'로 기재.
  - 외부 참조 링크는 소스 명시가 명확하지 않은 경우 미기재 또는 '확인 필요'로 표기.

## 3. 업무 Flow 변화 & 구현 방안 (Work Process)

- 샘플 Flow 문서 : AGENTS.md 및 README.md에 정의된 기본 에이전트 생성, 계획수립, 파일/쉘/서브에이전트 사용법

- 파일 네이밍

  - deepagents-cli.pyproject.toml
  - AGENTS.md
  - README.md

| 구분 | 내용 |
|---|---|
| 업무 Flow (AS-IS) | 제목: Deep Agents 현재 업무흐름<br>참여자: 사용자, Deep Agent SDK 및 CLI, LLM 모델 (OpenAI, Anthropic 등), 서브에이전트, 파일시스템, 쉘 환경<br>흐름: 1. 사용자 명령어 혹은 API 호출로 에이전트 실행 -> 2. 에이전트는 계획(write_todos) 수립 -> 3. 필요한 경우 파일 시스템 읽기/쓰기 수행 -> 4. 쉘 명령 실행 혹은 서브에이전트에 작업 위임 -> 5. 실행 결과를 종합해 응답 반환<br>출력: 계획된 작업 리스트, 파일 조작 결과, 쉘 명령 결과, 서브에이전트 작업 결과, 최종 응답 메시지 |
| 업무 Flow (TO-BE) | 제목: Deep Agents 목표 업무흐름<br>참여자: 사용자, Deep Agent SDK 및 CLI, LLM 모델, 확장 가능한 서브에이전트, 다양한 Sandbox (Daytona, Modal 등), 파일시스템, 쉘 환경<br>흐름: 1. 사용자 입력을 통해 에이전트 초기화 및 작업 시작 -> 2. 에이전트 내부에서 자동 요약·컨텍스트 관리 수행 -> 3. 계획 기술 강화 및 오류 감지 자동화 -> 4. 파일, 쉘, 서브에이전트 관리 기능 강화 및 병렬처리 최적화 -> 5. 결과 및 상태 실시간 모니터링 및 보고<br>출력: 향상된 작업 계획 및 진행 상태, 강화된 컨텍스트 처리 결과, 멀티샌드박스 연동 쉘 명령 출력, 서브에이전트 병렬 작업 결과, 자동화된 결과 리포트 |
| 구현 방안 | Core Agent: 플래닝, 작업 관리, 컨텍스트 조율 기능 수행, LLM과의 인터페이스 담당<br>Scheduler: 작업 스케줄링 및 워크플로우 관리, 서브에이전트 호출 조정<br>Worker (서브에이전트): 실제 작업 실행 (파일, 쉘, API 호출 등), 독립 컨텍스트로 작업 수행<br>External API 및 SDK: LLM 모델 접속, MCP, Agent Protocol 연동, 파일시스템 및 쉘 추상화 |

| 구현 단계 | 내용 |
|---|---|
| 1단계: 베이직 기능 구현 | 플래닝 및 파일시스템 조작 기능 완성<br>쉘 명령 실행 기능 내장<br>서브에이전트 단일 위임 구성 |
| 2단계: 확장 및 안정화 | 자동 요약 및 컨텍스트 관리 기능 보완<br>다중 서브에이전트 병렬 작업 지원<br>기본 샌드박스 및 권한 제어 구현 |
| 3단계: 프로덕션 배포 | LangSmith Deploy 연동 자동화<br>API 프로토콜 (MCP, A2A) 완전 지원<br>운영 모니터링 및 메모리 캐시 최적화 |
| 4단계: 고도화 및 최적화 | 다중 모델 지원 및 셀프 호스팅 강화<br>외부 샌드박스 (Daytona, Modal 등) 완전 통합<br>자동화된 테스트 및 린트/포맷 파이프라인 구축 |

## 5. Agent 동작 필수 파라미터 (Parameters)

| 시스템 | 파라미터 | 값 / 설명 |
|---|---|---|
| deepagents deploy | model | 사용자가 지정하는 LLM. OpenAI, Anthropic, Google 등 지원 |
| deepagents deploy | shell_allow_list | 실행 가능한 쉘 명령어 리스트 (기본: 'recommended,git,gh') |
| GitHub Actions | github_token | API 엑세스를 위한 토큰, 기본적으로 github.token 사용 |
| deepagents deploy | timeout | 최대 실행 시간 분 단위 (기본 30분) |
| 메모리 캐시 | enable_memory | 워크플로우 간 메모리 유지 여부 (기본 true) |
| 메모리 캐시 | memory_scope | 캐시 범위 (pr, branch, repo 중 선택, 기본 repo) |

## 6. 기타 참고사항 및 제약조건

- 실행 중 쉘 권한 및 허용 명령어 목록을 엄격히 관리해야 하며, 임의 명령 실행은 보안 위험 포함.
- 메모리 캐시 관련 권한 문제 및 데이터 유출 가능성 염두에 두어야 함.
- Beta 배포 상태로 API, 구성 및 동작이 변동 가능하므로 안정화 요구됨.
- 프로젝트 내 다중 패키지 및 모노레포 특성으로 버전 관리 및 릴리즈 프로세스 중요.
- 서브에이전트 작업 중 컨텍스트 충돌 및 리소스 관리 정책 필요.
- 외부 모델과 프로토콜 연동 시 키 관리, 토큰 제한 등에 대한 운영 고려 필요.

## Assumptions

- 사용자는 Python 환경에서 SDK 혹은 CLI를 사용해 에이전트를 제어한다고 가정.
- LangSmith 및 MCP 프로토콜은 정상적으로 작동하며, 추가 샌드박스 연동은 옵션 상태.
- GitHub 워크플로우 및 Actions를 통한 CI/CD가 운용 가능.
- 사용자 업무환경에 적합한 쉘 명령어 허용 리스트는 환경별로 조정 가능.
- Agent Skills 및 AGENTS.md 규격 기반 작업 instruction이 표준으로 준수됨.
- 문서 내 외부 링크 및 상세 구성은 최신 GitHub 저장소 및 공식 문서 참조 필요.
